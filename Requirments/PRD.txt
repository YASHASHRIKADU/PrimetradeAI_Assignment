📄 📌 PRODUCT REQUIREMENT DOCUMENT (PRD)
🏷️ Project Title

Smart Task Manager with Single Authentication & Role-Based Access

🎯 1. Objective

The objective of this project is to design and develop a secure, scalable REST API system with a basic frontend interface that allows users to manage tasks efficiently.

The system will implement:

Authentication using JWT
Role-based access control (Admin/User)
CRUD operations on tasks
Secure and scalable backend architecture
👥 2. Target Users
1. Regular Users
Register and login
Manage their own tasks
2. Admin Users
Manage all users’ tasks
Monitor system data
⚙️ 3. Key Features
🔐 3.1 Authentication System (Single System)
User registration
User login
Password hashing (bcrypt)
JWT-based authentication
👑 3.2 Role-Based Access Control (RBAC)
Roles:
user
admin
Middleware to restrict access
📝 3.3 Task Management (CRUD)
Features:
Create task
View tasks
Update task
Delete task
Task Fields:
Title
Description
Status (Pending/Completed)
Priority (Low/Medium/High)
Deadline
📊 3.4 Admin Features
View all users’ tasks
Delete any task
Monitor activity (basic)
🌐 3.5 Frontend Interface
Register/Login UI
Dashboard
Task list
Add/Edit/Delete task
Error & success messages
🧩 4. Functional Requirements
🔑 Authentication
System must allow users to register and login
JWT token must be generated after login
Protected routes must require token
🛡️ Authorization
Only admin can access admin routes
Users can only access their own data
🔄 Task Operations
Users can perform CRUD on their tasks
Admin can manage all tasks
⚠️ Error Handling
Proper status codes (200, 400, 401, 403, 500)
Meaningful error messages
📑 API Versioning
Use versioning: /api/v1/...
🗄️ 5. Non-Functional Requirements
🔒 Security
