📄 📌 TECHNICAL REQUIREMENT DOCUMENT (TRD)
🏷️ Project Title

Smart Task Manager with Single Authentication & Role-Based Access

🎯 1. System Overview

This system is a full-stack web application designed to manage tasks with:

Secure authentication (JWT)
Role-based authorization (Admin/User)
RESTful APIs
Scalable backend architecture

The backend will expose APIs consumed by a React-based frontend.

🏗️ 2. System Architecture
Architecture Type:
Client-Server Architecture
Flow:
Client (React) sends request
Server (Node.js/Express) processes logic
Database (MongoDB) stores/retrieves data
Response sent back to client
📂 3. Project Structure
backend/
 ├── controllers/
 ├── models/
 ├── routes/
 ├── middleware/
 ├── config/
 ├── utils/
 ├── app.js
 └── server.js
Description:
controllers/ → business logic

routes/ → API endpoints

🗄️ 5. Database Design
👤 User Schema
{
  name: String,
  email: { type: String, unique: true },
  password: String,
  role: { type: String, enum: ["user", "admin"] }
}
📝 Task Schema
{
  title: String,
  description: String,
  status: { type: String, enum: ["pending", "completed"] },
  priority: { type: String, enum: ["low", "medium", "high"] },
  deadline: Date,
  user: { type: ObjectId, ref: "User" }
}
🔐 6. Authentication Design
Method:
JWT-based authentication
Flow:
User logs in
Server validates credentials
JWT token generated
Token stored in client (localStorage)
Token sent in headers for protected routes
Header Format:
Authorization: Bearer <token>
🛡️ 7. Authorization Design
Role-Based Access Control (RBAC)
Roles:
user
admin
Implementation:
Middleware checks user role before granting access
🔌 8. API Design
🔐 Auth APIs
Register
POST /api/v1/auth/register
Input: name, email, password
Output: success message
Login
POST /api/v1/auth/login
Input: email, password
Output: JWT token
📝 Task APIs
Create Task
POST /api/v1/tasks
Auth required
Get Tasks
GET /api/v1/tasks
Returns only user’s tasks
Update Task
PUT /api/v1/tasks/:id
Delete Task
DELETE /api/v1/tasks/:id
👑 Admin APIs
Get All Tasks
GET /api/v1/admin/tasks
Delete Any Task
DELETE /api/v1/admin/tasks/:id
⚙️ 9. Middleware Design
1. Authentication Middleware
Verifies JWT token
Attaches user to request
2. Role Middleware
Checks if user is admin
Restricts unauthorized access
